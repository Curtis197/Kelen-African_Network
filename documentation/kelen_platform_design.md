# Kelen - Platform Design Document

## Core Philosophy

**"One mistake can end your career. Choose your clients. Keep every conversation."**

This is not a review platform. This is a **professional accountability system** where:
- Signals (contract violations) = permanent disqualification
- Recommendations (proven delivery) = career asset
- The standard is perfection, because the stakes are life savings

---

## The Two Completely Separate Systems

### System 1: VALIDATION (Free, Lookup-Only, Permanent)
**What it is:** The truth layer - documented project outcomes

**Access Model: LOOKUP BY NAME ONLY**
- Users can search by EXACT professional name/business name
- Purpose: "Check if someone you're about to work with has signals"
- NOT browsable/discoverable - you must know who you're looking for
- Like checking a criminal record - reactive, not proactive

**Components:**
1. **Recommendations** (positive validation)
   - Former client documents successful project completion
   - Evidence: contract + photos + timeline match
   - Pro can "link" these to their profile (claim ownership)
   - Visible ONLY when someone searches that specific pro's name
   
2. **Signals** (negative validation) 
   - Former client documents contract breach
   - Evidence: contract + photos + timeline divergence
   - Pro CANNOT remove these
   - ONE signal = permanent disqualification
   - Visible ONLY when someone searches that specific pro's name

**Rules:**
- ‚úÖ Always free to submit (users)
- ‚úÖ Always free to lookup by name (public)
- ‚úÖ Always verified before publication
- ‚ùå Cannot browse/discover professionals (that's Advertisement)
- ‚ùå Cannot be paid to remove
- ‚ùå Cannot be paid to hide
- ‚ùå Cannot be disputed away (only proven false)

**Pro's Role in Validation:**
- Can "link" recommendations to their profile (claim: "this is my work")
- Can respond to signals with counter-evidence (doesn't remove signal)
- Cannot prevent publication of verified evidence
- Profile exists but is NOT discoverable without Advertisement tier

---

### System 2: ADVERTISEMENT (CPM-Based Discovery)

**What it is:** Pay-per-visibility discovery system - completely independent of validation

**Critical Distinction:**
- **Validation = Lookup only** (search by exact name to check - FREE forever)
- **Advertisement = Discovery** (browse, find, be recommended - PAY per view)

## The CPM (Cost Per Mille) Model

**Professionals pay ‚Ç¨5 per 1000 profile views**

Not subscription. Not monthly fee. **Pay only for actual visibility.**

### How It Works:

1. **Professional adds credit to account:**
   ```
   Deposit ‚Ç¨50 ‚Üí Buy 10,000 profile views
   Deposit ‚Ç¨100 ‚Üí Buy 20,000 profile views
   Deposit ‚Ç¨500 ‚Üí Buy 100,000 profile views
   ```

2. **Set your cap (optional):**
   ```
   Option A: UNCAPPED
   - Profile stays visible until credit runs out
   - Views consume credit automatically
   - When balance hits zero ‚Üí removed from discovery
   - Can auto-reload to stay visible
   
   Option B: CAPPED (e.g., 5,000 views/month)
   - Profile visible up to cap
   - After 5,000 views ‚Üí removed from discovery until next month
   - Unused credit rolls over
   - Protects from burning through budget too fast
   ```

3. **Profile appears in discovery:**
   ```
   User searches: "Construction contractors in Abidjan"
   ‚Üí Pros with active credit appear in results
   ‚Üí Each profile view = ‚Ç¨0.005 deducted from pro's balance
   ‚Üí 1000 views = ‚Ç¨5 deducted
   ```

4. **When credit depletes:**
   ```
   Balance reaches zero ‚Üí
   - Profile REMOVED from discovery/browse
   - Still VISIBLE if someone searches exact name (validation)
   - Pro gets notification: "Credit depleted, add funds to stay visible"
   - Can reload immediately to resume discovery
   ```

## How "Linking" Recommendations Works

Since recommendations are submitted by clients, not pros, pros must **claim/link** them:

### Process:

1. **Client submits recommendation:**
   ```
   Former client: "I worked with Kouadio Construction"
   Evidence: Contract, completion photos, timeline
   Project: House in Abidjan, completed Dec 2024
   ```

2. **Recommendation goes to verification queue**
   - Kelen staff verify evidence exists
   - Verify contract is legitimate
   - Verify photos match project description

3. **Recommendation published (unlinked state):**
   ```
   ‚úì VERIFIED RECOMMENDATION
     Project: Residential construction, Abidjan
     Completed: December 2024
     Contractor mentioned: "Kouadio Construction"
     
     [Not yet linked to verified profile]
   ```

4. **Pro can claim/link it:**
   ```
   Kouadio Construction logs in ‚Üí
   See notification: "Recommendation mentions your business"
   Click "Link to my profile"
   Provide business verification (registration docs)
   
   ‚Üí Recommendation now appears on their profile
   ‚Üí Count increases: "4 verified recommendations"
   ```

### Why This System Works:

**Prevents fake recommendations:**
- Client submits first (with evidence)
- Pro can only claim what clients submitted
- Can't create recommendations for themselves

**Builds Gold status organically:**
- Each real project completion = potential recommendation
- Pro must maintain client relationships (they need them to submit)
- Natural incentive for quality work

**Allows pros to "opt in":**
- Recommendation exists even if pro doesn't claim it
- But claiming it builds their Gold status
- Unclaimed recommendations = lost opportunity
